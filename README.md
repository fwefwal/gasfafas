Москва 2025

## 1. Общие сведения

1.1. Полное наименование системы: Автоматизированная система управления кофейней «CoffeeFlow AS».  
1.2. Заказчик: ООО «КофеФлоу» (ИНН 7725123456).  
1.3. Исполнитель определяется по результатам открытого конкурса (государственный контракт №123 от __.__.2025).

  Начало — 10.12.2025  
  Окончание — 30.06.2026  

## 2. Назначение и цели создания системы

2.1. Назначение системы — автоматизация основных бизнес-процессов сети кофеен «КофеФлоу» (приём заказов, управление складом, учёт рабочего времени персонала, аналитика продаж).  

2.2. Основные цели создания системы:  
- сокращение времени обслуживания одного гостя с 120 до 45 секунд;  
- снижение потерь сырья на 25 % за счёт автоматизированного списания;  
- обеспечение возможности онлайн-заказа и оплаты через мобильное приложение;  
- предоставление руководству оперативной аналитики в режиме реального времени.  

## 3. Характеристика объектов автоматизации

3.1. Объекты автоматизации:  
- 5 кофеен в г. Москве;  
- центральный склад кофейных зёрен и расходных материалов;  
- колл-центр (приём онлайн-заказов);  
- мобильное приложение для гостей.  

3.2. Автоматизируемые процессы:  
- приём и выполнение заказов на месте и онлайн;  
- управление складскими остатками и автозаказ сырья;  
- учёт рабочего времени бариста и официантов;  
- формирование управленческой отчётности.

## 4. Требования к системе

### 4.1. Требования к системе в целом
- Архитектура: микросервисная, развёртывание в Docker + Kubernetes.  
- Надёжность: коэффициент готовности не менее 99,9 %.  
- Время восстановления после сбоя — не более 15 минут.  
- Защита информации: соответствие 152-ФЗ (УЗ-3, КЗ-3).  

### 4.2. Требования к функциям (задачам), выполняемым системой
| № | Подсистема            | Основные функции                                   |
|---|------------------------|-----------------------------------------------------|
| 1 | POS-терминал           | Приём заказов, печать чеков, безналичная оплата    |
| 2 | Онлайн-заказы          | Приём заказов через сайт и мобильное приложение    |
| 3 | Управление складом     | Учёт сырья, автозаказ при достижении min остатка   |
| 4 | Учёт рабочего времени  | Табель, расчёт ЗП, контроль графиков               |
| 5 | Аналитика и отчёты     | Дашборды в реальном времени, экспорт в Excel       |

### 4.3. Требования к видам обеспечения
- **Техническое**: планшеты на Android 11+, серверы в облаке Яндекс.Cloud.  
- **Программное**: PostgreSQL 16, Redis, Node.js 20, React Native.  
- **Информационное**: справочники номенклатуры, сотрудники, поставщики.  
- **Лингвистическое**: интерфейс на русском и английском языках.  
- **Организационное**: обучение 50 сотрудников, разработка 5 инструкций.  

### 4.4. Требования к режиму функционирования
Система должна работать в режиме 24×7×365 за исключением плановых работ (не более 4 часов в месяц, с 03:00 до 07:00 по МСК).
